**Master Concept Map --- The 5 Claude Surfaces**

Before diving into the individual videos, the unifying idea: Claude is not one product. It is a family of five surfaces, each optimized for a different mode of work. Knowing which surface to use for a given task is the meta-decision that determines whether your workflow succeeds or fizzles.

  --------------------------- ------------------------------------------------------------- -------------------------------------------------------------------------
  **Surface**                 **Best For**                                                  **When NOT To Use**
  Chat                        Quick questions, exploratory thinking, one-shot drafts        Anything involving local files, recurring work, or multi-step execution
  Cowork                      Multi-step tasks on local files, scheduled work, daily ops    Heavy code generation, terminal-level customization
  Claude Code                 Software engineering, agent teams, complex multi-file work    Non-coders who need a chat-like interface
  Managed Agents (Routines)   Cloud-hosted automation that runs while your laptop is off    Tasks needing local file access or interactive feedback
  API                         Programmatic integration, custom apps, fine-grained control   Most non-developer use cases --- overkill for daily ops
  --------------------------- ------------------------------------------------------------- -------------------------------------------------------------------------

**How They Work Together**

The professional pattern that emerges across all seven videos: Cowork is the operator\'s daily home base. Claude Code handles the deep technical work and agent teams. Routines run the recurring background processes that don\'t require your computer to be on. Chat is reserved for quick exploratory thinking. The API is for the rare case where none of the above fit.

**VIDEO 1**

**How to Use Claude Cowork To Do EVERYTHING For You (Complete Guide)**

**URL:** <https://www.youtube.com/watch?v=NZlblvXPJmI>

**Premise**

This video is positioned as the single complete reference guide for Cowork --- everything from first install to advanced multi-step automation. The host walks through every feature in order, with live demonstrations of each, ending with a working personal automation system.

**Section 1 --- What Cowork Actually Is**

Cowork uses the same agentic architecture that powers Claude Code, but is delivered inside Claude Desktop without requiring a terminal. Rather than responding to prompts one at a time, it takes on complex multi-step tasks and executes them on your behalf.

> *It feels much less like a back-and-forth and much more like leaving messages for a coworker.*

**Section 2 --- Critical Differences From Chat**

  ------------------------------------- ----------------------------------------------------
  **Limitation in Chat**                **How Cowork Solves It**
  20 files per conversation max         No file limit --- accesses local folders directly
  30MB per file ceiling                 No file size limit --- reads files in place
  Files must be uploaded to the cloud   Files stay on your machine
  Output appears in chat window only    Outputs land directly in your folder, ready to use
  Each conversation starts fresh        Persistent memory + project-scoped context
  ------------------------------------- ----------------------------------------------------

**Section 3 --- Requirements and Installation**

-   Paid plan required: Pro (\$20/mo), Max (\$100/mo), Team (\$30/user/mo), or Enterprise.

-   Platform: macOS or Windows (x64).

-   Active internet connection throughout the session.

-   Claude Desktop app installed.

-   Mode switch: Click the Cowork tab in the desktop app to enter the Cowork mode (it sits alongside Chat and Code as a top-level tab).

**Section 4 --- The First-Session Workflow**

1.  Click the Cowork tab to switch into Cowork mode.

2.  Switch the mode selector to \'Tasks\' for autonomous execution (vs. quick conversations).

3.  Describe the task you want Claude to complete, in plain language and outcome-focused.

4.  Review Claude\'s proposed approach --- Cowork generates a numbered plan before executing.

5.  Approve and let it run; outputs appear in the In-Progress and Artifacts panes as work progresses.

**Section 5 --- Folder Access**

Grant Cowork access to a specific folder on your computer. Critical safety guidance:

> *Avoid sharing your entire home directory. Create a dedicated project folder instead.*

This is one of the most important pieces of advice in the video. The blast radius of an error is dramatically smaller when Cowork is scoped to a single subfolder than when it has the run of your filesystem.

**Section 6 --- Connectors**

By default, Cowork only sees what\'s inside the granted folder. Connectors break that boundary by giving Cowork direct read/write access to external services.

-   Gmail --- read inbox, draft replies, triage.

-   Google Drive --- pull source files; write outputs back.

-   Notion --- read/write pages and databases.

-   Calendar --- query availability, schedule events.

-   100+ official connectors across business tools.

-   Custom MCPs available for unlisted tools.

**Section 7 --- Skills**

Skills are reusable instruction sets. Instead of re-explaining preferences every session, you create a skill once and reference it forever. This is presented in the video as the single highest-leverage feature of Cowork --- the difference between a chatbot and an operating system.

> *A skill is a custom workflow you teach Claude once and it runs forever --- you describe your preferences, your systems, and your specific needs, Claude learns them, then it just does it.*

**Section 8 --- Scheduled Tasks**

Once a skill works, schedule it. The /schedule command takes a prompt and a cadence, then runs the skill automatically with full access to everything you have connected.

Example demonstrated in the video: a skill that drafts client status updates, scheduled to run every Friday at 4 PM, pulling data from Gmail and Notion connectors and delivering a completed draft to a designated folder.

**Section 9 --- The Big Mistake Most Users Make**

> *The biggest mistake people make with Cowork is treating it like ChatGPT with folder access.*

The host frames this as the central trap: people install Cowork, use it for one back-and-forth task, conclude it is just chat with file access, and abandon it. The reality is that Cowork is more like leaving instructions for a coworker. The interaction pattern is delegation, not conversation.

**Section 10 --- File Structure Lives Or Dies**

> *Cowork lives or dies by how you structure your files. Build core files that teach Cowork who you are, what you\'re working on, and how you like things done.*

This recurring theme --- the structural foundation under everything --- is the host\'s strongest claim. The hours invested in creating context files, organizing your folder taxonomy, and defining skills are not preparatory work; they are the work. Everything Cowork does is downstream of those foundational artifacts.

**Key Takeaway**

This video is the long-form reference. If you watch only one Cowork video as a beginner, it is comprehensive enough to set you up for everything else.
